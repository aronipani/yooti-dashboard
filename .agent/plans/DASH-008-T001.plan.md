## Implementation steps

1. Write MSW handlers for all 6 API endpoints in `msw-handlers.ts`
2. Write failing tests for each hook: loading state, success, error with retry
3. Create `api-client.ts`:
   - axios instance with baseURL from `import.meta.env.VITE_API_BASE_URL`
   - Never hardcode localhost or port numbers
4. Create each hook using @tanstack/react-query:
   - `useCurrentMetrics(projectId)` → GET /metrics/current
   - `useSprintList(projectId)` → GET /metrics/sprints
   - `useSprintDetail(projectId, sprintNum)` → GET /metrics/sprint/{num}
   - `useTrends(projectId, metric?)` → GET /metrics/trends
   - `useStoryDetail(projectId, sprint, storyId)` → GET /metrics/story/{id}
   - Each hook: { data, isLoading, isError, error, refetch }
5. Create `SprintSelector.tsx`:
   - Uses useSprintList to populate dropdown
   - onChange triggers parent callback with selected sprint_num
   - Loading skeleton while fetching
   - Error state with retry button
6. Write axe-core accessibility test for SprintSelector
7. Test: 404 response → isError true, retry button calls refetch

## Constraints
- VITE_API_BASE_URL from .env — never hardcoded
- All hooks tested with MSW — no real API calls
- Loading skeleton during fetch
- Error state with retry for every hook
- axe-core test in every component test
- No direct localhost URLs in component code
