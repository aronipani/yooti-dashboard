## Implementation steps

1. Write failing tests for all 4 AC scenarios
2. Create `ingest_validator.py`: validate project_id, sprint_num, stories array presence and types
3. Create `ingest_metrics.py` handler:
   - Parse event["body"] with json.loads (never assume dict)
   - Validate input → 400 on failure
   - Call repository.batch_write_sprint() for all entities
   - Detect first-write vs re-ingest (check if sprint summary exists) → 201 or 200
   - Catch RepositoryError → 500 INTERNAL_ERROR (no internal detail)
   - Return dict with statusCode, headers (CORS), body
4. boto3 client at module level
5. All error responses use standard error shape from ARCH-DASHBOARD.md Section 5

## Constraints
- event["body"] always parsed with json.loads
- Error responses never expose internal detail
- Handler returns dict — never raises
- All tests use @mock_aws
