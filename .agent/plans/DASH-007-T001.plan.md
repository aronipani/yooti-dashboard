1. Write failing tests for file readers (gate files, evidence packages, escalations, corrections, logs)
2. Create `file_readers.py`:
   - `read_gate_files(agent_dir)` → list of gate records per story
   - `read_evidence_packages(agent_dir)` → test results, coverage per story
   - `read_escalations(agent_dir)` → escalation records
   - `read_corrections(agent_dir)` → correction records
   - `read_audit_logs(agent_dir)` → log entries

3. Write failing tests for metric calculator
4. Create `metric_calculator.py`:
   - `compute_sprint_summary(stories, gates, evidence, escalations, corrections)` → dict matching sprint summary attributes from ARCH-DASHBOARD.md
   - Computes: cycle_time_avg, sprint_completion_pct, agent_exec_hrs_avg, human_wait_hrs_avg, deploy_frequency, coverage_new_code_avg, mutation_score_avg, regression_rate_pct, constitution_pass_pct, escalation_rate_pct, iteration_avg, correction_rate_pct, gate_rejection_rates, escalation_breakdown, phase_avg_hrs
   - `compute_story_metrics(story_data, gates, evidence)` → dict matching story metrics attributes
