## Files in Scope:
- CREATE: `scripts/deploy-frontend.sh`
- CREATE: `.github/workflows/deploy-frontend.yml`

## Implementation steps:
1. Create deploy-frontend.sh:
   - Read S3_BUCKET and CF_DISTRIBUTION_ID from environment variables
   - Run `npm run build` in frontend/dashboard/
   - `aws s3 sync dist/ s3://$S3_BUCKET --delete`
   - Set cache headers: `--cache-control max-age=31536000` for assets/, `--cache-control no-cache` for index.html
   - `aws cloudfront create-invalidation --distribution-id $CF_DISTRIBUTION_ID --paths "/*"`
2. Create GitHub Actions workflow:
   - Trigger: push to main
   - Steps: checkout, setup-node, npm install, npm run build, configure AWS credentials (from secrets), run deploy script
   - AWS credentials via `aws-actions/configure-aws-credentials` with GitHub secrets

## Constraints:
- S3 bucket name and CloudFront ID from Terraform outputs — not hardcoded in script
- AWS credentials from GitHub secrets or OIDC — never hardcoded
- --delete flag on s3 sync to remove stale files
- index.html always no-cache to ensure latest version served
