## Implementation steps

1. Write failing tests for insight alert computation
2. Create `insight_engine.py` with `compute_alerts(sprint_summary) -> list[dict]`:
   - BOTTLENECK: any phase_avg_hrs value exceeds threshold
   - HUMAN_VS_AGENT: human_wait_hrs_avg > agent_exec_hrs_avg by margin
   - CONSTITUTION_GAP: constitution_pass_pct < 95%
   - QUALITY_DROP: coverage_new_code_avg < 90% or regression_rate_pct > 5%
   - REGRESSION_SPIKE: regression_rate_pct > 10%
   - ESCALATION_SPIKE: escalation_rate_pct > 15%
   - DORA_GAP: deploy_frequency < target
3. Each alert: `{ type, severity, message, metric, value, threshold }`
4. Modify ingest handler to:
   - After batch write, compute alerts from sprint summary
   - Build CURRENT#SNAPSHOT with all KPI fields + alerts + last_ingested_at + TTL (90 days)
   - Call repository.put_current_snapshot()
5. Test: ingest with values triggering each alert type → alerts present in snapshot

## Constraints
- Alerts computed at ingest time, not query time (D3)
- Insight alert shape matches ARCH-DASHBOARD.md Section 3
- All thresholds configurable via constants (not env vars)
- All tests use @mock_aws
