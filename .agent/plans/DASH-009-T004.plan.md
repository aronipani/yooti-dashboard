## Implementation steps

1. Write failing tests for ValueTab
2. Create `ValueTab.tsx`:
   - KPI cards: stories_shipped, sprint_completion_pct, ac_coverage summary
   - Delivery trend chart: stories_shipped over sprints
   - Completion rate trend chart
   - Uses useCurrentMetrics + useTrends hooks
3. Write failing tests for StoryTable
4. Create `StoryTable.tsx`:
   - Table columns: story_id, type, status, cycle_time, iterations, coverage, escalation
   - Sortable by any column
   - Row click triggers onStorySelect callback
5. Write failing tests for StoryDetailPanel
6. Create `StoryDetailPanel.tsx`:
   - Slide-out panel or modal
   - Gate timeline: G1 → G2 → G3 → G4 → G5 with timestamps and reviewers
   - Phase durations: visual bar for each phase
   - Escalation log: list of escalations with type and resolution
   - Evidence summary: test results, coverage
   - Close button
   - Uses useStoryDetail hook (fetches on open)
7. Write failing tests for StoryDrilldownTab
8. Create `StoryDrilldownTab.tsx`:
   - StoryTable showing all stories in selected sprint
   - StoryDetailPanel opens on row click
   - Uses useSprintDetail for story list
9. axe-core test in every component test
10. Responsive: table scrolls horizontally on mobile, detail panel full-width on mobile

## Constraints
- Story click triggers API call — not pre-loaded
- All data from hooks — no hardcoded values
- Chart.js for charts, HTML table for story list
- Responsive at 320px, 768px, 1280px
- axe-core in every test
